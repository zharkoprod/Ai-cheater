02 · Что реально известно про СПб

9 районов, которые мы можем подтвердить

Конкретные числа PM2.5 по каждому району Петербург не публикует — только сводки по 25 станциям. Но качественные характеристики «чисто / норма / грязно» есть у Северо-Западного УГМС, ГГО им. Воейкова, экопортала Комитета по природопользованию и Wikipedia. Вот что подтверждается источниками.

Честная оговорка: мы не выдумываем числа. Цифры PM2.5 на карточках — это годовые средние по городу СПб (УГМС, 2024), переложенные на коэффициент района по качественным рейтингам. Точные значения по каждому району и есть та задача, которую решает бот — собирает их из 5 источников в момент запроса.

04 · Почему не IQAir и не Яндекс

Шесть причин выбрать нас

У каждого аналога — половина решения. Мы первые, кто собрал всё в один Telegram-чат с понятным языком.

Бесплатно. Навсегда.

Один запрос — 0 ₽. Полная стоимость прототипа на 10 000 пользователей — 12 тыс. ₽ в год.

В Telegram, без приложений

Никаких скачиваний, никаких регистраций, никаких СМС. Открыл бот — задал вопрос — получил ответ.

5 параметров, не 1

Не только воздух. Шум, вода, зелень, события — единый индекс в %, понятный с первого взгляда.

Российские нормы

СанПиН 1.2.3685-21, Р 2.1.10.1920-04, рекомендации ВОЗ 2021. Не американский AQI на ваши лёгкие.

Подписка на район

Один раз настроил — бот сам пингует утром, если воздух ухудшился. Можешь успеть забрать ребёнка из секции.

Открытая методика

Никакого «чёрного ящика». Формула, веса и источники открыты — можешь проверить любую цифру.

06 · Частые вопросы

FAQ

А это правда бесплатно?

Да. Один запрос для пользователя — 0 ₽. Вся годовая инфраструктура (VPS, сертификаты, бэкапы, API-лимиты) для прототипа на 5–10 тыс. человек стоит около 12 тыс. ₽ в год — это покрывается грантом или личным карманом разработчика, не пользователем.

Откуда бот берёт данные?

Из пяти открытых источников: OpenAQ (мировая база замеров воздуха), Северо-Западное УГМС / ГГО им. Воейкова (государственные данные по СПб), OpenStreetMap (шум и зелень по картам), Sentinel-2 (спутниковая съёмка озеленения), Росприроднадзор + МЧС (текущие экологические события). Платных API внутри нет.

Зачем мне это, если можно открыть Яндекс.Погоду?

Яндекс показывает один обобщённый индекс качества воздуха по городу. Это полезно, но не отвечает на вопрос «что у меня прямо сейчас в Невском». Чистый Стандарт идёт глубже: 5 параметров, разбивка по районам, российские ПДК и понятная рекомендация — «гулять или не гулять».

Бот хранит мою геолокацию?

Только если ты сам нажал «Подписаться на район» — тогда бот запоминает выбранный район (не точные координаты), чтобы слать тебе пуш при превышении порога. Отписаться можно одной командой /unsub. Соответствие ФЗ-152.

А если бот покажет, что воздух плохой — что делать?

В карточке всегда есть конкретный совет: закрыть окна, включить очиститель, надеть FFP2, отменить пробежку. Для родителей и аллергиков — расширенные рекомендации по кнопке «Я с ребёнком».

Когда уже будет реальный бот в Telegram?

Это курсовой проект СПбГМТУ 2026. Сейчас защищается концепция и алгоритм. Реализация в виде прототипа — следующий шаг, по предварительной оценке 2–3 месяца силами небольшой команды. Если хочешь поучаствовать или просто получать апдейты — напиши автору (контакты в футере).

Формула расчёта интегрального риска EВзвешенная сумма 5 частных индексов · опора на Р 2.1.10.1920-04
+

Каждый частный индекс I нормирован на [0; 1] как отношение фактического воздействия к ПДК (СанПиН 1.2.3685-21). При превышении ПДК — штрафной коэффициент 1.2.

E = ( w₁·I_air + w₂·I_water + w₃·I_noise + w₄·I_green + w₅·I_events ) · 100 %

Веса: w₁=0.40 (воздух), w₂=0.20 (вода), w₃=0.15 (шум), w₄=0.15 (озеленение), w₅=0.10 (события). Воздуху наибольший вес: по данным ВОЗ — до 70% антропогенного риска здоровью горожанина.

Пример расчёта (Центральный, СПб, 12.03.2025): PM2.5=18, PM10=35, NO₂=0.11, SO₂=0.02. Max(C/ПДК)=1.10 → I_air=(1.10·1.2)/4=0.33. I_water=0.15, I_noise=0.55, I_green=0.60, I_events=0.10. Итог: E=34.5% → УМЕРЕННЫЙ.
