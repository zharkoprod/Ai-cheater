# Нейро-Читер — лендинг

Готовый лендинг для деплоя на **GitHub Pages**. Один HTML-файл, всё внутри (CSS, JS, картинки в base64), плюс одно видео в `images/`.

---

## Что внутри папки

```
сайт-github/
├── index.html                  ← сам лендинг (606 KB)
├── images/
│   └── timofey-bonus.mp4       ← бонус-видео Тимофея (39 МБ)
├── .nojekyll                   ← пустой файл, чтобы GitHub не прогонял через Jekyll
└── README.md                   ← этот файл
```

5 остальных картинок (Чингиз, claude-collage, petya-correspondence, alan-correspondence, petya-card) уже встроены **прямо в index.html** как base64 — отдельно их грузить не надо.

## Как развернуть на GitHub Pages (5 минут)

### 1. Создай репозиторий

1. Зайди на https://github.com → справа сверху нажми **+** → **New repository**
2. **Repository name** — любое короткое имя, например `neuro-cheater` (английскими, без пробелов)
3. **Public** (для GitHub Pages нужно публичное)
4. **Add a README file** — НЕ ставь галочку (у нас свой README уже есть)
5. Жми **Create repository**

### 2. Залей файлы

Самый простой способ — через интерфейс GitHub без терминала:

1. На странице нового пустого репозитория нажми **"uploading an existing file"** (синяя ссылка в центре)
2. Перетащи в окно браузера **ВСЁ содержимое этой папки** (`index.html`, папку `images/`, `.nojekyll`, `README.md`)
3. Подожди пока загрузится (видео 39 МБ, минута-две)
4. Внизу нажми **Commit changes**

### 3. Включи GitHub Pages

1. В репозитории сверху → **Settings**
2. Слева в меню → **Pages**
3. В блоке **Source** выбери **"Deploy from a branch"**
4. **Branch** → **main** (или **master**), папка **`/ (root)`**
5. Жми **Save**
6. Через 30-60 секунд вверху появится сообщение: **"Your site is live at https://ТВОЙ-НИК.github.io/neuro-cheater/"**

Это и есть твой адрес лендинга. Можешь скидывать в сторис, рекламу, бота.

## Подключить свой домен (опционально)

Если хочешь чтобы было не `username.github.io/...` а, например, `neuro-cheater.ru`:

1. Купи домен у любого регистратора (reg.ru, beget, namecheap)
2. В DNS-настройках домена добавь A-записи на адреса GitHub:
   ```
   185.199.108.153
   185.199.109.153
   185.199.110.153
   185.199.111.153
   ```
3. В Settings → Pages в поле **Custom domain** впиши свой домен → Save
4. Подожди 10-30 минут пока DNS прорастёт
5. Поставь галочку **Enforce HTTPS** (нужно подождать пока GitHub выпустит сертификат — обычно час)

## Что нужно проверить после деплоя

Открой свой URL в браузере и в Chrome DevTools (F12 → Console). Должно быть:

- ✅ Заголовок выезжает построчно, потом три профессии (копирайтер / SMM-щик / дизайнер) по очереди зачёркиваются
- ✅ Цифры в hero-блоке наматываются от нуля (20+ млн, 15, 77к, 2/5)
- ✅ Sticky-навигация сверху осталась на месте при скролле, есть прогресс-бар
- ✅ Видео Алана / Тимофея / Чингиза проигрываются (YouTube embed)
- ✅ Бонус-видео Тимофея mp4 играет
- ✅ Кнопка "Скопировать" в подарке копирует промпт
- ✅ Якоря "Забрать" скроллят к секции тарифов

Если что-то не работает — пришли скриншот + URL.

## Что нужно поправить перед публикой

В файле `index.html` найди два места (через **Ctrl+F**):

1. **`href="#"`** в секции тарифов (блок 18) — **2 кнопки оплаты** ведут в пустоту. Замени `#` на свои реальные ссылки оплаты:
   - "Забрать за 2 990 ₽" → ссылка на оплату базового
   - "Забрать с разбором за 14 900 ₽" → ссылка на оплату премиум

Все остальные `href="#tariffs"` (в кейсах, в P.S.) — НЕ трогай, это якорь к секции тарифов.

## Как потом обновлять

Если захочешь что-то поменять в лендинге — два варианта:

**Вариант A (через GitHub в браузере):**
1. Открой репозиторий → нажми на `index.html` → карандашик "Edit this file"
2. Внеси правки → внизу **Commit changes**
3. Через минуту сайт обновится

**Вариант B (попроси меня):**
Просто скажи что поменять — я пришлю новый `index.html`, ты его загрузишь поверх старого (там же где загружал в первый раз: **Add file** → **Upload files** → перетянул → Commit).

## Размер репо

Всего ~40 МБ (39 МБ — видео Тимофея, 0.6 МБ — HTML). У GitHub лимит 100 МБ на файл и до 1 ГБ на репозиторий — мы спокойно в рамках.

Если в будущем добавишь больше тяжёлых видео — стоит выложить их на YouTube как unlisted и встроить через iframe (как у тебя сейчас сделано для Алана/Тимофея/Чингиза). Это и быстрее грузится у пользователей.
